# Termidesk Support Bot

Интеллектуальный помощник (RAG-система) для технической поддержки продукта Termidesk. 
Бот анализирует официальную документацию (PDF), находит релевантные фрагменты с помощью векторного поиска и генерирует точные ответы на вопросы пользователей через веб-интерфейс и Telegram.

## 🚀 Основные возможности
*   **Инкрементальная индексация:** Индексатор умеет автоматически находить новые/измененные PDF файлы и добавлять их в базу знаний без переиндексации старых.
*   **Гибридный поиск (Hybrid Search):** Комбинация семантического поиска (Qdrant) и лексического поиска (Whoosh) для максимальной точности нахождения нужных глав.
*   **Оценка уверенности (CRAG):** Бот умеет оценивать найденную информацию (evaluator). Если ответ не найден, бот честно об этом скажет или запросит уточнение, избегая галлюцинаций.
*   **Web & Telegram Frontend:** Удобный веб-чат (FastAPI + WebSockets) и интеграция с Telegram.
*   **Курируемый кэш:** Ответы бота могут быть оценены пользователями (Лайк/Дизлайк). Одобренные ответы попадают в Redis кэш, что моментально ускоряет обработку частых вопросов.
*   **Глубокий мониторинг (ELK + Grafana):** Логи всех запросов и их задержки (latency) собираются через Logstash в Elasticsearch и отображаются на дашбордах Grafana.

## 📂 Структура репозитория
*   `app/`, `indexing/`, `frontend/` — исходный код FastAPI, AI-индексатора и UI.
*   `data/` — золотая база знаний. **Внимание:** векторная БД весит >1 ГБ, поэтому она не хранится в Git! Смотрите инструкцию по установке ниже.
*   `docs/` — **Итоговый отчет и финальная презентация** по проекту (по требованию куратора).
*   `monitoring/` — конфиги ELK (Elasticsearch, Logstash, Kibana), Prometheus и Grafana.

## 🛠 Установка и Запуск (Мини-инструкция для куратора)

Мы подготовили проект для развертывания "в один клик". Мы используем заранее собранный Docker-образ (`suslik000ilya/termidesk_bot:latest`), чтобы вам не пришлось компилировать зависимости.

### Шаг 1: Скачивание Базы Знаний
Чтобы бот мог отвечать на вопросы, ему нужна проиндексированная векторная база документов.
1. Перейдите во вкладку **Релизы** (Releases) в этом репозитории на **GitFlic**.
2. Скачайте архив `knowledge_base.zip`.
3. Распакуйте содержимое архива в папку `data/` в корне проекта (с заменой файлов).

### Шаг 2: Запуск (Production Deploy)
Вам понадобятся: установленный Docker и файл `.env` (обратитесь к автору проекта за `.env` с ключами).
1. Убедитесь, что порты 8000, 9200, 5601, 3000 не заняты.
2. В терминале в корне проекта выполните команду:
   ```bash
   docker compose -f docker-compose.prod.yml up -d
   ```
3. Откройте **http://localhost:8000** в браузере — веб-интерфейс бота готов к работе! Дашборды мониторинга доступны на **http://localhost:3000** (Grafana) и **http://localhost:5601** (Kibana).

---

### Альтернатива: Для разработчиков (Build from Source)
Если вы хотите изменять исходный код и тестировать логику бота:
1. Выполните шаги 1 и 2 из инструкции выше.
2. Запустите классический `docker-compose.yml`, который соберет образ локально и пробросит папки:
   ```bash
   docker compose up --build -d
   ```

## 🔄 Как обновлять базу знаний?
Если вы положили новые PDF-файлы в папку `data/pdfs/версия/`:

1. Подключитесь к запущенному контейнеру бота и запустите индексатор:
   ```bash
   docker compose exec bot python -m indexing.indexer
   ```
2. Индексатор сам найдет новые файлы, пропустит старые и обновит базы `Qdrant` и `Whoosh`. 
3. *Отказоустойчивость:* Если процесс прервется (пропадет интернет), просто запустите команду заново. Скрипт сам подчистит "огрызки" и продолжит работу с того файла, на котором остановился.
