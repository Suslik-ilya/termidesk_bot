# Termidesk Support Bot

Интеллектуальный помощник (RAG-система) для технической поддержки продукта Termidesk.
Бот анализирует официальную документацию (PDF-файлы), находит релевантные фрагменты с помощью векторного поиска и генерирует ответы на вопросы пользователей через веб-интерфейс и Telegram.

## Основные возможности

* **Инкрементальная индексация** — автоматическое обнаружение новых или измененных PDF-файлов и добавление их в базу знаний без переиндексации существующих данных.
* **Гибридный поиск (Hybrid Search)** — комбинация семантического поиска (Qdrant) и лексического поиска (Whoosh) для повышения релевантности выдачи.
* **Оценка уверенности (CRAG)** — встроенный механизм оценки найденной информации. При отсутствии релевантного контекста система запрашивает уточнение, минимизируя вероятность галлюцинаций.
* **Интерфейсы взаимодействия** — веб-чат (FastAPI + WebSockets) и Telegram-бот.
* **Курируемый кэш** — система оценки ответов пользователями (лайк/дизлайк). Одобренные ответы сохраняются в Redis-кэш для ускорения обработки типовых запросов. Управление кэшем осуществляется через админ-панель.
* **Универсальность моделей** — поддержка работы как с облачными LLM через OpenAI-совместимый API, так и с локально развернутыми моделями (параметры подключения настраиваются через `.env`).
* **Мониторинг (ELK + Grafana)** — сбор метрик и логов запросов через Logstash в Elasticsearch с визуализацией на дашбордах Grafana.

## Структура репозитория

```
termidesk_bot/
├── app/                        # Backend-приложение (FastAPI)
│   ├── adapters/               # Адаптеры: Telegram-бот, WebSocket, админ-панель
│   ├── agent/                  # AI-агент (граф CRAG)
│   │   └── nodes/              # Узлы графа: роутер, ретривер, генератор, оценщик и др.
│   ├── services/               # Сервисы: LLM-клиент, Redis, Qdrant, Whoosh, кэш, метрики
│   ├── templates/              # HTML-шаблоны (админ-панель)
│   └── main.py                 # Точка входа FastAPI
├── config/                     # Конфигурация приложения (settings.py)
├── data/                       # Данные (не хранятся в Git, см. раздел «База знаний»)
│   ├── pdfs/                   # Исходные PDF-документы (по подпапкам версий: 5.1/, 6.1/)
│   ├── qdrant/                 # Хранилище векторной БД Qdrant
│   └── whoosh_index/           # Индекс лексического поиска Whoosh
├── docs/                       # Документация, отчеты, презентации
├── frontend/                   # Веб-интерфейс чата (index.html)
├── indexing/                   # Скрипт индексации PDF-документов
├── monitoring/                 # Конфигурации стека мониторинга
│   ├── grafana/provisioning/   # Дашборды и источники данных Grafana
│   ├── logstash/pipeline/      # Pipeline конфигурация Logstash
│   └── prometheus.yml          # Конфигурация Prometheus
├── .env.example                # Шаблон переменных окружения
├── docker-compose.yml          # Конфигурация для разработки (сборка из исходников)
├── docker-compose.prod.yml     # Конфигурация для production (готовый образ)
├── Dockerfile                  # Сборка Docker-образа приложения
└── requirements.txt            # Python-зависимости
```

## Требования

* **Docker** и **Docker Compose** (версия 2+)
* Свободные порты: `8000`, `3000`, `5601`, `6333`, `6379`, `9090`, `9200`
* Минимум 4 ГБ оперативной памяти (Elasticsearch и модель эмбеддингов требуют ресурсов)

## Развертывание

### 1. Клонирование репозитория

```bash
git clone https://github.com/Suslik-ilya/termidesk_bot.git
cd termidesk_bot
```

### 2. Настройка переменных окружения

Скопируйте шаблон и заполните его своими значениями:

```bash
cp .env.example .env
```

Откройте файл `.env` в текстовом редакторе и укажите необходимые параметры:

| Переменная | Описание |
|---|---|
| `LLM_API_KEY` | API-ключ для доступа к LLM (OpenAI-совместимый провайдер) |
| `PROXY_API_URL` | Базовый URL API-провайдера. Для локальной модели укажите адрес вашего сервера (например, `http://localhost:1234/v1`) |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота (получается через @BotFather). Необязателен, если не нужен Telegram-интерфейс |
| `JIRA_URL`, `JIRA_USER`, `JIRA_API_TOKEN` | Параметры интеграции с Jira для эскалации обращений. Необязательны |

Остальные переменные (адреса Redis, Qdrant, Logstash и пр.) уже настроены на значения по умолчанию и не требуют изменений при стандартном развертывании через Docker Compose.

### 3. База знаний

Проиндексированная векторная база документов имеет большой объем и не хранится в системе контроля версий. Существует два способа получить рабочую базу:

**Способ А. Скачать готовую базу (рекомендуется):**

Перейдите во вкладку **Releases** данного репозитория, скачайте архив `knowledge_base.zip` и распакуйте его содержимое в директорию `data/` с заменой файлов. После этого в директории `data/` должны появиться заполненные подпапки `pdfs/`, `qdrant/`, `whoosh_index/` и файл `processed_files.json`.

**Способ Б. Собрать базу самостоятельно:**

Положите PDF-документы в директорию `data/pdfs/`, разделив их по подпапкам версий (например, `data/pdfs/5.1/`, `data/pdfs/6.1/`). Индексация запускается после развертывания сервисов (см. раздел «Индексация документов» ниже).

### 4. Запуск сервисов

Проект поддерживает два варианта запуска:

#### Вариант А. Готовый образ (Production)

Использует предварительно собранный образ из Docker Hub. Не требует локальной компиляции.

```bash
docker compose -f docker-compose.prod.yml up -d
```

#### Вариант Б. Сборка из исходников (Development)

Собирает образ локально из исходного кода. Исходники пробрасываются внутрь контейнера, что позволяет вносить изменения в код без пересборки.

```bash
docker compose up --build -d
```

После запуска дождитесь, пока все контейнеры перейдут в состояние `healthy`. Проверить статус можно командой:

```bash
docker compose ps
```

## Использование

### Веб-интерфейс чата

Откройте в браузере:

```
http://localhost:8000
```

Введите вопрос по документации Termidesk и получите ответ от бота. Используйте кнопки лайка и дизлайка, чтобы оценить ответ. Если хотите сменить тему, нажмите на кнопку `Очистить диалог` или продолжайте писать в чат, а бот догадается о смене темы и предложит очистить контекст. Если вас не устраивает ответ и вы хотите отправить заявку в поддержку, то напишите боту о своём желании или согласитесь обратиться в поддержку, если бот сам не справился с поиском ответа и предложил эскалацию. Если вас интересует вопрос по конкретной версии документации, то прямо в вопросе укажите номер версии или дождитесь, когда бот сам спросит интересующую версию, и назовите её.

### Telegram-бот

Если в `.env` указан `TELEGRAM_BOT_TOKEN`, бот запускается автоматически вместе с сервером. Найдите вашего бота в Telegram и начните диалог. Функционал Telegram-бота идентичен веб-версии, общайтесь так же, как и в веб-чате. Отличается только сброс контекста: вместо кнопки вы можете написать команду `/reset`.

### Админ-панель (управление кэшем)

Доступна по адресу:

```
http://localhost:8000/admin/
```

На этой странице отображаются ответы-кандидаты в кэш (те, которые получили положительную оценку от пользователей). Администратор может одобрить, отредактировать или удалить записи.

### REST API

Приложение предоставляет следующие эндпоинты:

| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/api/v1/chat` | Отправить вопрос боту. Тело: `{"session_id": "...", "message": "..."}` |
| `POST` | `/api/v1/feedback` | Отправить оценку ответа. Тело: `{"message_id": "...", "session_id": "...", "feedback": "like"}` |
| `POST` | `/api/v1/reset` | Сбросить контекст диалога. Тело: `{"session_id": "..."}` |
| `GET` | `/admin/` | Админ-панель управления кэшем |

### Мониторинг

| Сервис | Адрес | Описание |
|---|---|---|
| Grafana | `http://localhost:3000` | Дашборды с бизнес-метриками (количество запросов, latency LLM, эскалации). Логин по умолчанию: `admin` / `admin` |
| Kibana | `http://localhost:5601` | Просмотр и анализ логов, собранных через Logstash |
| Prometheus | `http://localhost:9090` | Метрики приложения (используется Grafana в качестве фронтенда) |

## Индексация документов

Для добавления новых документов в базу знаний:

1. Поместите PDF-файлы в директорию `data/pdfs/[версия]/` (например, `data/pdfs/6.2/`).
2. Запустите скрипт индексации внутри контейнера:

```bash
docker compose exec bot python -m indexing.indexer
```

Скрипт автоматически обнаружит только новые или измененные файлы и обновит векторную (Qdrant) и лексическую (Whoosh) базы данных. Состояние индексации фиксируется в файле `data/processed_files.json`, что позволяет безопасно прерывать и возобновлять процесс.
